Setiap fitur di Web Helper juga berfungsi sebagai REST API mentah — tanpa API key, menerima semua metode HTTP, dan bisa dipanggil langsung dari bot, aplikasi, atau skrip apa pun.

Pengenalan

Web Helper punya dua sisi yang saling terhubung: tampilan dashboard yang bisa dipakai langsung dari browser, dan endpoint REST API mentah di balik setiap fitur. Base URL mengikuti domain tempat kamu men-deploy proyek ini, misalnya https://domainkamu.com.

Pola umum endpoint download/search:

Pola URL
/api/{platform}/{search|download}?q={kata_kunci_atau_link}&format={opsional}

Semua endpoint menerima metode HTTP apa pun (GET, POST, PUT, dll) — parameter query q dan format tetap dibaca dari query string terlepas dari metode yang dipakai.

Format Respons (Envelope)

Semua endpoint — tanpa kecuali — membungkus hasilnya dalam struktur JSON yang sama:

Struktur envelope 
{
  "creator": "XYCoolcraft",
  "developer": "XYCoolcraft",
  "version": "v1.0.0",
  "status": true,
  "ping": "482ms",
  "time": "2026-07-02T10:41:02.118Z",
  "provider": "y2mate",
  "result": {}
}
FieldTipeKeterangan
statusbooleantrue jika berhasil, false jika seluruh provider fallback gagal
pingstringWaktu proses permintaan dari server menerima hingga membalas
timestring (ISO 8601)Waktu server saat respons dibuat
providerstringNama provider/sumber yang berhasil dipakai (untuk endpoint dengan fallback)
resultobjectData hasil — struktur berbeda tiap endpoint, lihat bagian masing-masing di bawah
errorstringHanya muncul kalau status false — pesan gabungan semua provider yang gagal dicoba

Job & Polling (untuk proses yang lama)

Proses download video/audio dan olah gambar butuh waktu, dan hosting gratis (seperti Vercel Hobby) memotong request yang berjalan lebih dari beberapa detik. Untuk mengakalinya, endpoint download pada YouTube, Spotify, dan Image Tools tidak langsung mengembalikan hasil — melainkan sebuah jobId yang harus di-polling.

1

Panggil endpoint download seperti biasa. Responnya langsung kembali dalam hitungan milidetik berisi result.jobId.

2

Panggil GET /api/job/{jobId} setiap 2–3 detik untuk mengecek status.

3

Selama proses berjalan, result.status bernilai "pending". Terus polling sampai berubah.

4

Begitu selesai, result.status menjadi "completed" dan data lengkap (termasuk downloadUrl) ikut disertakan.

Contoh alur job
// 1) Mulai proses
GET /api/yt/download?q=https://youtu.be/dQw4w9WgXcQ&format=720
→ { "status": true, "result": { "jobId": "m3x9k2ab", "poll": "/api/job/m3x9k2ab" } }

// 2) Polling
GET /api/job/m3x9k2ab
→ { "status": true, "result": { "status": "pending" } }
   ... ulangi setiap 2-3 detik ...
→ { "status": true, "result": { "status": "completed", "title": "...", "downloadUrl": "https://..." } }

Job disimpan di memori server selama 30 menit sejak dibuat, lalu otomatis dihapus. TikTok, Facebook, Instagram, Twitter/X, dan CapCut tidak memakai job/polling — hasilnya langsung dikembalikan di respons pertama karena sumbernya cukup cepat.

▶️ YouTube

Deteksi otomatis: kalau q berupa link youtube.com/youtu.be, langsung diproses sebagai video. Kalau berupa teks bebas, dipakai sebagai kata kunci pencarian.

ANY/api/yt/search?q={kata_kunci}

Mengembalikan array berisi banyak video (bukan cuma satu) lengkap dengan thumbnail, channel, durasi, dan jumlah views — cocok dirender sebagai daftar hasil pencarian.

ParamWajibKeterangan
qYaKata kunci pencarian
{
  "result": [
    { "videoId": "dQw4w9WgXcQ", "title": "...", "channel": "...", "url": "https://youtu.be/dQw4w9WgXcQ", "duration": "3:32", "views": "1.4B views", "thumbnail": "https://..." },
    { "videoId": "...", "title": "...", "...": "... (bisa puluhan hasil)" }
  ]

ANY/api/yt/download?q={link_atau_kata_kunci}&format={format}

format: 144–1080 untuk video mp4, atau 128/256/320 untuk audio mp3. Default 720. Mengembalikan jobId — lihat bagian Job & Polling.

Bahasa & Routing

Setiap halaman dashboard memakai path /{halaman}/{kode_bahasa}, contoh /home/en, /downloader/youtube/ja. Kode bahasa yang didukung: id, en, es, fr, de, pt, ru, ar, hi, ja, ko, zh.

Endpoint API (/api/...) tidak memakai kode bahasa — responnya selalu JSON mentah apa adanya supaya konsisten dipakai sebagai REST API dari luar.

Kode Error

HTTP StatusArti
400Parameter wajib tidak ada (misal base64 kosong di Image Tools)
404jobId tidak ditemukan atau sudah kedaluwarsa (lebih dari 30 menit)
502Semua provider fallback gagal — cek field error untuk detail pesan tiap provider

Batasan & Timeout

Di hosting serverless gratis seperti Vercel Hobby, satu request maksimal berjalan sekitar 10 detik sebelum dipotong platform — walau axios di server ini di-set timeout jauh lebih lama. Karena itu proses berat (download, olah gambar) dipecah jadi job asinkron: request awal cuma beberapa milidetik untuk membuat job, lalu klien polling /api/job/{jobId} sampai selesai — pola ini juga dipakai persis oleh halaman dashboard-nya sendiri.

Untuk hosting VPS/dedicated tanpa batas waktu serverless, jalankan lewat install.sh yang sudah disediakan — tidak ada perbedaan kode, hanya cara deploy yang berbeda.
